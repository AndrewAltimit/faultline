# build-maps

One-shot Node script that derives `site/js/app/generated-regions.js` from
public-domain country polygons.

## Why

Faultline scenarios render region polygons as lightweight [lon, lat] rings so
the WASM-side renderer stays dependency-free. Hand-drawn polygons are fast but
inaccurate; this script applies Ramer–Douglas–Peucker simplification to the
`datasets/geo-countries` GeoJSON so the checked-in module stays small while
still tracking real borders.

## Re-building

```bash
curl -L -o tools/build-maps/countries.geojson \
  https://raw.githubusercontent.com/datasets/geo-countries/master/data/countries.geojson
node tools/build-maps/build.mjs
```

The source GeoJSON (~14 MB) is **not** checked in — fetch it on demand when
re-generating. The output lives at `site/js/app/generated-regions.js` and is
committed.

## Source data

- `datasets/geo-countries` (CC0 / ODC-PDDL) — simplified Natural Earth 10m
  admin-0 boundaries.
- The US macro-region polygons in `site/js/app/us-regions-geo.js` are
  hand-authored and **not** regenerated by this script.
